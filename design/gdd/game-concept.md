# Game Concept: 烦恼粉碎机 (Worry Crusher)

*Created: 2026-05-20*
*Status: Draft*

---

## Elevator Pitch

> 一款移动端解压小游戏：你把烦恼写下来喂进一台可爱的卡通粉碎机，拉一下摇杆，机器轰隆隆把烦恼打碎——然后从下方吐出一个意外的小宝贝（一朵花、一只蝴蝶、一颗小星星）。所有产物收藏在你的玩具货架上，烦恼变成了你的纪念品。

**10 秒测试**：✅ 任何人 10 秒内能想象到玩什么——一台粉碎机、写字、拉杆、看小东西出来。

## Core Identity

| Aspect | Detail |
| ---- | ---- |
| **Genre** | Casual / 解压工具 / 收集型 mood toy |
| **Platform** | Mobile（iOS 优先 → Android 跟进）|
| **Target Audience** | 18-35 岁，工作/学习压力大、需要稳定情绪出口的人；详见 Target Player Profile |
| **Player Count** | 单人 |
| **Session Length** | **1-3 分钟 / 次**（极短，适合碎片场景）|
| **Monetization** | MVP 永久免费，无广告无内购；v1.5+ 可选 cosmetic 一次性付费（皮肤、装饰）|
| **Estimated Scope** | Small（3-4 周 MVP / 6-8 周 v1.0，solo）|
| **Comparable Titles** | Pokémon Sleep（拟人陪伴）、Florence（极简情绪叙事）、Forest（解压仪式感）、纸鱼/AHA（情绪 App 形态）|

## Core Fantasy

> **"我的每一个烦恼都没有白白消失——它被一台靠谱的小机器认真处理掉了，变成了一个会陪伴我的小宝贝。烦恼成了我的收藏。"**

这款游戏的情绪承诺不是"消灭"烦恼，而是**重新定义烦恼的命运**：你不是把负面情绪丢进垃圾桶，而是交给一个会眨眼睛的小工匠，它郑重其事地把这件事处理掉，并且**回馈给你一件意外的、温柔的东西**。

为什么这个比"日记 App"或"冥想 App"更有效：日记是把情绪"封存"，冥想是把情绪"放下"——这款游戏让玩家**亲眼看见烦恼的转化过程**，是一个完整的"输入 → 仪式 → 转化 → 收获"的闭环。

---

## Unique Hook

> **像解压 App + Pokémon 图鉴 + 扭蛋机，AND ALSO 你写下来的烦恼真的被一台有人格的机器"消化"了，变成你独有的纪念品。**

通过 "and also" 测试：
- 像解压 App，**AND ALSO** 解压完之后还能留下漂亮的东西
- 像扭蛋机，**AND ALSO** 你投入的不是钱，是你今天的烦恼
- 像 Pokémon 收集图鉴，**AND ALSO** 每一只小宝贝背后都对应你过去某一刻的真实情绪

这个 Hook 影响玩法，不只是表层包装——它决定了**没有失败、没有难度、没有 PVP，只有"输入情绪 → 见证转化 → 收藏陪伴"的核心节奏**。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Submission**（放松、无压力）| **1（主）** | 不限次数、无 timer、无任务、无失败；机器永远在那里 |
| **Sensation**（感官愉悦）| 2 | 震动 + 齿轮音 + 粉碎可见 + 揭晓"叮咚" + 摇杆触感 |
| **Discovery**（盲盒+剪影揭晓）| 3 | 剪影+颜色暗示 → 点击揭晓的微戏剧 |
| **Fantasy**（人格化机器）| 4 | 机器是有名字、有性格的小角色 |
| **Expression**（你的烦恼）| 5 | 玩家输入文字，输入内容是私人的、被尊重的 |
| **Narrative**（剧情）| N/A | 无 |
| **Challenge**（技巧）| N/A | 无难度，无失败 |
| **Fellowship**（社交）| N/A | 不上传、不分享、不联机 |

### Key Dynamics（期望玩家自然产生的行为）

- 玩家会**自发形成"打开 → 写烦恼 → 粉碎 → 看产物 → 关闭"的固定情绪卫生仪式**（早上 / 通勤 / 睡前各一次）
- 玩家会**在不同情绪下使用同一个出口**——焦虑、委屈、愤怒、失落都丢进同一台机器
- 玩家会**偶尔翻回收藏馆**，看以前的产物，回忆当时的情绪——货架变成"情绪日记"
- 玩家会**因为好奇心回访**："今天会出什么？"
- 玩家会**和机器小角色建立单向情感关系**（"它在等我"）

### Core Mechanics（实际实现的系统）

1. **Text Input**：文字输入（最少 5 字，自动拒绝纯重复符号）
2. **Lever Pull**：摇杆拉动（触觉反馈 + 音效 + 机器抖动动画）
3. **Shred Process**：粉碎过程（碎屑在机器内部可见旋转）
4. **Silhouette Reveal**：剪影揭晓（黑色 + 一抹颜色暗示 → 点击 → 弹跳填色）
5. **Shelf Collection**：货架式收藏馆（本地存档、按时间或类型排列）

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | 想什么时候用就什么时候用、想写什么就写什么、想看哪个就看哪个；零强制 | **Core** |
| **Competence** | 不靠技能挑战，而是"我每天好好处理自己的情绪"的 **self-care 胜任感** | Supporting |
| **Relatedness** | 和机器角色的单向陪伴关系；机器永远在那里、永远靠谱 | Supporting |

### Player Type Appeal (Bartle)

- [x] **Achievers** — 弱：货架累积、产物种类越来越多有"小成就感"，但不靠"必须集齐"驱动
- [x] **Explorers** — **强（主要类型）**：每次都好奇"会出什么"，剪影揭晓机制完美服务这类型
- [ ] **Socializers** — 明确不为：无社交、无分享
- [ ] **Killers/Competitors** — 明确不为：无对抗、无排行

### Flow State Design

- **Onboarding（前 60 秒）**：第一次打开 → 机器自我介绍（一句话）→ 引导写第一个烦恼 → 第一次粉碎**强制出特别产物**（比如金色小心，给"原来这游戏长这样"的强印象）
- **Difficulty scaling**：N/A，这是放松游戏，没有难度曲线
- **Feedback clarity**：触觉 + 音效 + 视觉三重反馈，每个动作都有清晰回响
- **Recovery from failure**：N/A，没有失败状态

## Core Loop

### Moment-to-Moment (30 秒)

```
1.  打开 App → 看到待机的机器（眨眼/打盹）
2.  点击机器嘴/上方喂纸口 → 空白纸条弹出
3.  打字写下今天的烦恼（≥5 字，5-15 秒）
4.  写完，纸滑入机器嘴里 → 机器表情变"好奇/准备工作"
5.  ⭐ 拉摇杆（核心爽感）：震动 + 齿轮声 + 机器抖动 + 内部光闪 (3 秒)
6.  粉碎过程：碎屑在透明腔体里旋转 (2 秒)
7.  ⭐ 底部"叮"，剪影从出口推出（黑色 + 一抹颜色暗示）
8.  ⭐ 玩家点击剪影 → 砰一下填色 + 弹跳 → 揭晓产物
9.  产物停留在托盘上（可拿起观察）
10. 几秒后/玩家手动 → 产物飞入货架收藏馆

总循环：约 22-30 秒
```

**为什么 30 秒循环本身就有趣**：核心爽感 = 摇杆的物理反馈 + 揭晓的微剧场。这两件事不依赖进度、不依赖排名、不依赖故事——拉一次就爽一次，第 100 次和第 1 次同等有效。

### Short-Term (5-15 分钟)

玩家可能连续粉碎 1-3 个烦恼（今天积累的不同事情），偶尔遇到**不常见产物**（15% 概率）或**稀有产物**（5% 概率）。每次都有完整剪影流程——仪式感永远在线。

### Session-Level (1-3 分钟实际，最多 5 分钟)

一次完整使用 ≈ 1-3 个烦恼 + 偶尔翻一下货架。不鼓励超长 session——这是工具，不是娱乐时长产品。

### Long-Term Progression

| 阶段 | 体验 |
| ---- | ---- |
| 第 1-3 天 | 解锁基础 8 种产物中的大部分 |
| 第 1-3 周 | 偶尔遇到稀有产物（仅 v1.0+）|
| 第 1 个月+ | 货架摆满，进入"维护期"——产物开始重复，但仪式感不变 |
| 第 2 个月+ | （v1.5）解锁机器皮肤 / 季节限定产物，给重复用户新刺激 |

### Retention Hooks

- **Curiosity（好奇）**：每次"会出什么"的轻微好奇心
- **Investment（投入）**：货架累积了一段时间后，"我这个月攒了 N 个" 的归属感
- **Social**：N/A（反支柱）
- **Mastery**：N/A（无技能挑战）
- ⚠️ **不依赖 retention 任务/签到/连击**——这是反支柱

## Game Pillars

### Pillar 1：触感先行（Tactile First）
粉碎机的物理反馈是这款游戏的灵魂。震动、齿轮声、机器抖动必须扎实——视觉漂亮可以让步，触感不能让步。

*Design test*：如果在"更精致的产物动画" vs "更扎实的摇杆震动"之间选——**选触感**。

### Pillar 2：每一次都是一场小剧场（Every Pull Is a Theatre）
玩家就算粉碎第 100 次同一个产物，从摇杆 → 剪影 → 揭晓的微戏剧都必须完整。仪式感不能被"省事"或"提速"侵蚀。

*Design test*：如果有人提议"重复产物跳过剪影"——**拒绝**。仪式感优先于效率。

### Pillar 3：收藏不焦虑（Collection Without Pressure）
货架累积是奖励，但永远不能让玩家感到"还差 N 个就集齐"的压力。稀有 = 锦上添花，永远不 = 任务。

*Design test*：任何 UI 或文案让玩家感到"必须集齐"的压力——**砍掉**。

### Pillar 4：可爱但有情感重量（Cute But Weighted）
卡通可爱是表皮，但玩家输入的烦恼内核是真实的。不能因为"放松游戏"就让一切变成糖水甜剧。

*Design test*：当玩家写"我妈妈病重"这种重内容时，机器**不应该跳萌做表情包，也不应该装哀悼**——它应该用同样靠谱工匠的姿态完成工作。轻重得当。

### Pillar 5：不限次但永远有意义（Unlimited But Meaningful）
你可以一天粉碎 100 次，体验不退化；你三天不打开，游戏也不催你。这是工具不是 retention loop。

*Design test*：不设每日任务、不设连击奖励、不设"你 3 天没用了"推送。

### Anti-Pillars (What This Game Is NOT)

- **NOT 战斗游戏**：粉碎不是攻击，是处理。机器是炼金师，不是武器。
- **NOT 集齐成就驱动**：没有"图鉴 100% 解锁奖励"这种激励。
- **NOT 日常签到 / 留存任务**：工具游戏不该有手游运营 loop。
- **NOT 付费抽卡**：稀有产物只靠时间和运气，**不能用钱加速**。这是放松游戏的底线。
- **NOT 云同步 / 不上传烦恼内容**：玩家写的话只在本地，**隐私是默认设置**。

## Visual Identity Anchor

### 方向名：**Cozy Mechanical（暖系机械）**

**一句话视觉规则**：
> "一台会眨眼的工业级粉碎机，立在一片乳白色的留白里。"

### 三条视觉原则

1. **机器扎实，环境留白** —— 机器本身有重量感（厚壳、可见螺丝、使用磨痕），但永远放在宽阔的乳白色背景里。主体重，环境轻。
   - *测试*：背景里**不要**放任何装饰（盆栽、桌面、海报）。环境越空，机器越有存在感。

2. **机器有表情，但表情是"克制的工匠"** —— 不是 emoji 那种夸张，是 Wall-E / 豆豆兵那种**只用眼睛和眉毛**就能传情的表情。
   - *测试*：表情库不超过 8 种，每一种都辨识度高，但**都不夸张**。

3. **产物要小、要不可名状的可爱** —— 不做"大花"或"高清宝石"，要做**"一只手就能捧住的迷你扭蛋玩具"**。
   - *测试*：所有产物视觉重量比机器小至少 5 倍。

### 色彩哲学

- **基底**：乳白 / 牛奶白
- **机器主色**：奶油色 / 黄绿色 / 浅原色（80/90 年代家电色卡——梦幻西游怀旧感落地）
- **产物**：每个产物有自己鲜亮的纯色（饱和度温和，不刺眼）
- **稀有揭晓时**：金粉色 + 闪光颗粒（仅此时使用）

### 视觉参考

- *Untitled Goose Game* 的极简留白
- *双影奇境* 的克制色彩 / 唯美场景
- *Wall-E* 的人格化机器（仅靠眼睛和眉毛传情）
- 扭蛋机的工业感
- 90 年代办公室碎纸机的怀旧质感
- 梦幻西游"玩具柜"的童年珍藏感

**这个 Anchor 是 Art Bible 的种子**——下游所有美术决策（机器外观、产物风格、UI、字体、动效）都必须服务这个 Anchor，或回来挑战它。

## Risks and Open Questions

### Design Risks

- **核心循环可能很快"看穿"**：8 个产物 + 不限次数 → 玩家可能 30 分钟内就见过全部，新鲜感掉得快。**缓解**：MVP 阶段重点验证"机械触感本身是否爽到值得重复"，而非靠产物数量。
- **文字输入可能被滥用**：玩家可能写"asdf"或乱按只为看动画——稀释"情绪倾诉"的仪式感。**缓解**：最少 5 字 + 拒绝纯重复符号 + 重复内容 24 小时内不再触发新粉碎。
- **"轻重得当"是难题**：当玩家写重内容（亲人去世、抑郁）时，可爱机器跳萌会显得不尊重。**缓解**：MVP 阶段机器保持中性平静，不主动跳萌；后续可考虑情感识别。

### Technical Risks

- **iOS / Android 震动反馈体验差异大**：iPhone Taptic Engine 细腻，Android Vibrator 粗糙——同样代码体验天差地别。**缓解**：iOS 优先，Android 跟进时降级实现。
- **音效专业度门槛**：业余音效会让"机械爽感"完全失效。**缓解**：预算分配——音效 ≥ 美术；考虑外包 / 高质量免费素材包改 / AI 音效工具辅助。
- **第一款游戏 + 几周 + 移动端打包流程不熟**：Godot iOS 出包流程对新手有坑。**缓解**：提前 1 周开始端到端打包验证，不要留到最后。

### Market Risks

- **解压 App / Mood Toy 赛道竞品多**：Calm、Headspace、Pokémon Sleep、AHA、纸鱼……都已存在。**缓解**：定位差异化清晰——我们是**"工具 + 游戏"混合**，不是纯 App，也不是纯游戏。差异点是机械触感 + 收藏盲盒。
- **"烦恼粉碎机"名字偏负面**：可能让人误以为是"发泄类"暴力游戏。**缓解**：考虑英文副标题或视觉传达暖意，或在 App Store 描述中突出"chill / cozy"关键词。

### Scope Risks

- **几周时间预算 + 第一款 + 美术工作量** = 经典超时陷阱。**缓解**：MVP 严守"1 机器 + 8 产物"上限；美术超时 → 用免费素材包或 AI 辅助过渡。
- **触觉反馈如果调不好，整个游戏失去灵魂**——这是单点失败风险。**缓解**：开发第 1 周就建端到端 prototype 验证触感，不通过则重新评估方向。

### Open Questions

- **机器叫什么名字？性格细节是什么？** → 移交 `/design-system` 中"角色设计"环节决定
- **MVP 8 个产物具体是什么？** → 移交 `/design-system` 中"产物系统"环节决定
- **触觉反馈具体曲线如何设计？** → 移交 prototype 阶段 (`/prototype`) 验证
- **音效从哪来？** → 移交 `/setup-engine` 后讨论音频管线
- **是否需要在隐私页明确"烦恼内容不上传"？** → 移交 release-checklist 阶段

## MVP Definition

**Core hypothesis**：
> 玩家在 22-30 秒的核心循环（打字 → 拉摇杆 → 剪影揭晓 → 货架累积）里能持续获得**情绪释放**和**惊喜感**，并且**愿意第二天再回来**。

**MVP 要回答的唯一问题**：核心循环爽不爽？不爽则砍掉所有后续投入。

### Required for MVP

1. **一个主画面**：机器在屏幕中央 + 货架入口图标 + 上方输入区
2. **文字输入功能**：最少 5 字限制，自动拒绝纯重复符号
3. **摇杆拉动动画 + 触觉反馈**（iOS 优先优化）
4. **粉碎过程动画**：碎屑在机器内部可见旋转
5. **剪影揭晓机制**：黑色 + 颜色暗示 → 点击 → 填色弹跳
6. **8 种产物**：80% 常见 + 20% 不常见，**MVP 不做稀有层**
7. **货架式收藏馆**：本地存档，按时间排列
8. **本地存档系统**：所有数据存本地，不上传

### Explicitly NOT in MVP

- ❌ 颜色变体（产物多种颜色）—— v1.0 加
- ❌ 5% 稀有产物层 —— v1.0 加
- ❌ 机器皮肤系统 —— v1.5
- ❌ 季节限定产物 —— v1.5
- ❌ 设置页 / 偏好 —— v1.0
- ❌ 云同步 / 账户 —— 永不（反支柱）
- ❌ 任何社交分享 —— 永不（反支柱）
- ❌ 任何付费/广告 —— 永不（反支柱）

### Scope Tiers

| Tier | Content | Features | Timeline (solo) |
| ---- | ---- | ---- | ---- |
| **MVP** | 8 产物，无稀有，本地 | 核心循环 + 触觉反馈 + 货架 | **3-4 周** |
| **v1.0** | 20 产物 + 5 颜色变体 + 5% 稀有层 | + 设置页 + 触觉调优 + 音效 polish | 6-8 周（累计）|
| **v1.5** | + 机器皮肤 + 季节限定 | + cosmetic 付费 + 装饰货架 | 上线后增量更新 |
| **Full Vision** | 50+ 产物 + 多机器皮肤 + 节日活动 | + 分享单个产物截图（可选） | 上线 + 3-6 月 |
